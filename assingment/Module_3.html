Q-5. What are different types of lists in HTML?

Lists are used to put together relevant pieces of information so that they are easily readable and clearly related to one another. Lists are essential components in contemporary web development and are regularly used for both navigation and general content.

The three list types

  • unordered list

    • -used to arrange a collection of connected elements without regard to order.

  • ordered list

    • - used to show name/value pairs, such as words and definitions, in a

  • description list

    • - certain order and to group a set of related objects.

Unordered list

When a group of elements can be arranged in any order, unordered (bulleted) lists are utilized. A shopping list is an illustration:

  • milk
  • bread
  • butter
  • coffee beans

Unordered list Markup

Unordered lists are composed of one set of <ul> </ul> tags followed by one or more sets of <li> </li> tags:

<ul>
   <li> bread </li>
   <li> cofee beans </li>
    <li> milk </li>
   <li> butter </li>
</ul>

Ordered lists

In order to present a list of things that must be in a particular sequence, ordered (numbered) lists are utilized. An illustration would be a recipe:

  • 1.assemble the ingredients
  • 2. Combine ingredients.
  • 3. Ingredients should be placed in a baking dish and baked for one hour.
  • 4. Take the oven out.
  • 5. 10 minutes should be given for standing.
  • 6. Serve

Ordered lists can be displayed in a variety of ways. Decimal numbers are typically the default in browsers, however there are additional options as well:

  • Letters
    • Lowercase ascii letters (a, b, c…)
    • Uppercase ascii letters (A, B, C…).
  • Numbers
    • Decimal numbers (1, 2, 3…)
    • Roman numerals (i, ii, iii…)

Ordered list markup

Ordered lists use one set of <ul> </ul> tags wrapped around one or more sets of <li> </li> tags:

<ul>
   <li> assemble the ingredients </li>
   <li>Combine ingredients. </li>
    <li> Ingredients should be placed in a baking dish and baked for one hour. </li>
   <li> Take the oven out. </li>
   <li> 10 minutes should be given for standing. </li>
   <li> Serve </li>
</ul>

Description lists

Definition lists, which were renamed to description lists in HTML5, are lists that link particular names and values. Examples include the elements in an ingredient list and their brief descriptions, the writers of articles, and the years that they won competitions. There must be at least one name and at least one value in each pair regardless of how many name-value groupings you have.

Since they are versatile, description lists allow you to link several values to a single name or vice versa.
You may demonstrate each meaning of the word "coffee," for instance, one after the other:

coffee
a coffee-based beverage that has been ground and roasted
an iced coffee
a social event where medium- to dark-brown coffee is drank

Description list markup

One or more groups of <dt> </dt> (name) and <dd> </dd> (value) tags are wrapped around one or more sets of <dl> </dl> (description list) tags. It is required to match at least one dt with at least one dd, with the dt always coming first in the source sequence.

A simple description list of single names with single values would look like this:

<dl>
   <dt> Name </dt>
   <dd>Value </dd>
    <dt> Name </dt>
   <dd> Value </dd>
</dl>

Q-6. What is the ‘class’ attribute in HTML?

An element's classnames can be set using the class attribute in HTML. You can use CSS to operate with the given class name and point to a stylesheet.

This is an example of how to use the class attribute in HTML.

Example

Live Demo <!DOCTYPE html>

</html>

Q7. What is the difference between the ‘id’ attribute and the ‘class’ attribute of HTML elements?

HTML id Attribute:

The document is identified by the id attribute, which is a special identifier. JavaScript and CSS both employ it to carry out a specific task for a distinct element. The id attribute in CSS is denoted by the # sign followed by id.

Syntax:

<element id= "id_name">
In CSS Stylesheet:
#id_name {
// CSS Property
}

Example:

html <!DOCTYPE html>
<html>

<head>
<title>
HTML id attribute
</title>
<style>
#geeks{
color: green;
font-size: 25px;
}
</style>
</head>
<body style="text-align: center">
<h1>Noman</h1>
<p> id="Noman">Welcome to Noman</p>
<p>A Computer Science portal for Noman</p>
</body>

</html>

Output:

Welcome to Noman

HTML class Attribute:

For an HTML element, the class attribute is used to specify one or more class names. Any HTML element can use the class attribute. CSS and JavaScript can both use the class name to perform specific actions on items that have the specified class name. The "." symbol is used with the class name in CSS stylesheets.

Syntax:

<element class= "class_name">
In CSS Stylesheet:
.class_name {
// CSS Property
}

Example:

html <!DOCTYPE html>
<html>

<head>
<style>

Q-8. What are the various formatting tags in HTML?

There are different types of HTML tags are used to format the appearance of the text on your web page.

1. Make headings using the <h1> to <h6> elements:

Every document has a heading at the top. Your headings might be different sizes. There are 6 levels of headings, with h1 being the biggest and most significant heading and h6 being the smallest. An illustration of the code for each headline size is provided here:

<h1> Level 1 Headline </h6>
<h2> Level 2 Headline </h2>
<h3> Level 3 Headline </h3>
<h4> Level 4 Headline </h4>
<h5> Level 5 Headline </h5>
<h6> Level 6 Headline </h6>

2. Establish a paragraph using the <p> element:

It is possible to divide your content into paragraphs using the <p> element. As seen in the sample below, each paragraph of text should be separated by an initial <p> and a closing </p> tag.

<p> Here is a paragraph of text. </p>
<p>Here is a second paragraph of text. </p>
<p> Here is a third paragraph of text. </p>

3. Making Line Breaks With the <"br> Element

Everything that follows the <br/>element always begins on the next line. This tag is an example of an empty element because there is nothing in between the opening and closing tags.

There is a space between the letters br and the forward slash in the <br/> element. Older browsers will have problems rendering the line break if you don't include this space.

4. Bold - <b> </b>:

Like in a word processor, the text between the tags will be bold and stand out from the text surrounding it.

5. Italic - <i> </i> :

Italics display the text at a slight slant and operate similarly to a word processor.

6. Underline - <u> </u>:

The same as underlining in a word processor, once more. HTML links are already underlined, thus there is no need for the additional tag.

7. Strike-out - <strike> </strike>:

creates a line through the text's center, erasing it. Used frequently to indicate that a text is out-of-date and irrelevant.

8. Preformatted Text - <pre> </pre>:

Any content between the pre tags will display in the browser like it would in a text editor, including spaces, carriage returns, and punctuation.

9. Source Code - <code> </code>:

The text is presented in a fixed-width typeface, similar to tt, and is frequently used to display source code. To display all tags on this website, I combined it with stylesheets.

10. Typewriter Text - <tt> </tt>:

The text has a fixed-width typeface, giving the impression that it was typewritten. For instance: The <tt> and </tt> tags are used to format this text.

11.Block Quote - <blockquote> </blockquote>:

identifies a long quotation, which is presented with a margin to the left of the block quote that is very broad.

12. Small - <small> </small>:

Use the small tag to render text somewhat smaller than the surrounding text without of having to set a font size. helpful for outlining the "fine print."

13. Centre - <center> </center>:

A helpful tag that, as stated, centers anything between the tags

14. Emphasis - <em> </em>:/h4>

used to highlight text, which is often italicized but may vary depending on your browser.

15.Strong Emphasis - <strong> </strong>:

Text that is used to emphasize material more generally appears in bold, but your browser may display it differently.
