Cascading Style Sheet (CSS) is a style sheet that provides a set of style rules for defining the layout of HTML documents. Q.12 • How to maintain the CSS specifications?

With the introduction of CSS3, the CSS Specification is the same, but it now takes a different approach.The Specification provides extensive algorithms, code snippets, and tabular data in addition to defining how CSS attributes should be implemented by browser vendors.

Additionally, the specification contains:

  • The language's grammar and data types
  • Explanation of CSS Selectors in detail
  • the process for putting values on attributes
  • The Cascade (CSS's "C").
  • The process of inheritance
  • e.g., The Box Model

Some of these topics are briefly and simply presented, while others are explained in considerable length.

The Specification also outlines how to target particular media, such as print or screen, and how stylesheets can be incorporated into your web publication.

The CSS specification before CSS3 was a single specification, whereas CSS3 is divided into modules, which are independent specifications that can be worked on by different author(s) at different speeds. For example, we have selector level 3 specification, CSS colour 4, CSS backgrounds, and so on. All of these modules are under the CSS3 umbrella, however some are updates to CSS2.1 and some are brand-new.

Q.13 • What are the ways to integrate CSS as a web page?

1. Inline-Style

The first way to incorporate CSS into HTML is using a technique known as inline-styling. When CSS rules are added directly into HTML elements (tags), this is referred to as inline styling.

For instance, suppose I want to alter an element's text colour:

<h1 style="color: red">Test Headline</h1>

The new rule (in this case, text colour) will then be applied to the h1 element:

Test Headline

2. Internal CSS

The internal CSS method is the second approach to include CSS in HTML. This method requires the usage of an HTML tag called "style" (not "style attribute"), between which we can place our CSS selectors and rules:

<style>
h1 {
color: red;
}
</style>
<body>
<h1>Test Headline</h1>
</body>

Test Headline

3. External CSS

The ideal practise is to keep HTML and CSS separate. In actual programming, HTML, CSS, and JavaScript must all be kept in distinct files and imported as needed. This method facilitates code maintenance and enhances readability.

To employ this method, we must first create individual CSS files with the.css extension and then connect them to HTML.

We could, for instance, make a CSS file called index.css. Our CSS rules are written in the index.css file:

h1 {
color: red;
}

Then we can import index.css to HTML with a <link> tag like below:

<head>
<link rel="stylesheet" type="text/css" href="index.css">
</head>
<body>
<h1> Test Headline </h1>
</body>
